// Judging Criteria

How Projects
Are Evaluated

Innovation & Idea

Originality, creativity, uniqueness of the solution

GitHub Repository

Commits, collaboration, README quality, proper usage

Technical Execution

Complexity, tech stack choices, code quality

Problem Relevance

Real-world impact, clarity of problem statement

Functionality

Working prototype, features implemented

Presentation

Demo clarity, explanation quality, Q&A handling

Judging Rounds
1
Execution Phase ~50% project complete, GitHub activity checked
2
Final Evaluation MVP ready + final presentation
Elimination Criteria
  • No GitHub repository submission
  • Incomplete or non-working project
  • Plagiarism or copied work
  • Team member absent during evaluation
  • No portal registration
  • No new members after registration

  • Best use of GitHub

  • Good UI/UX design
